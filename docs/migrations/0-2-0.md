# Migration notes for 0.2.0

## Qdrant Quadlet observer parameters

The Qdrant Quadlet validation entry points now accept observer trait objects:

```rust
pub fn validate_checked_in_qdrant_quadlet(
    observer: &dyn QdrantQuadletObserver,
) -> Result<(), QdrantQuadletError>;

pub fn validate_qdrant_quadlet(
    contents: &str,
    observer: &dyn QdrantQuadletObserver,
) -> Result<(), QdrantQuadletError>;
```

Existing callers that pass a concrete observer by reference do not need to
change:

```rust
validate_qdrant_quadlet(contents, &TracingQdrantQuadletObserver)?;
```

Callers that store observers behind trait objects can now pass those references
directly:

```rust
let observer: &dyn QdrantQuadletObserver = &TracingQdrantQuadletObserver;
validate_qdrant_quadlet(contents, observer)?;
```

Tests that do not need telemetry can continue to pass `&()` because `()`
implements `QdrantQuadletObserver` as a no-op observer.
